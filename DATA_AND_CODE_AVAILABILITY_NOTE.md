# Data and code availability statement

Shareable derived data supporting Figs. 1-7, Tables 1-4 and Supplementary Table S1 are provided in `data/`. The package includes figure source tables, derived geospatial layers, grid and district model outputs, network and threshold diagnostics, manual-validation summaries, source-ledger fields, metadata dictionaries and review-screen tables. A manuscript-element crosswalk is provided in `DATA_CODE_FIGURE_TABLE_CROSSWALK.md` and `data_code_figure_table_crosswalk.csv`, specifying which files underlie each figure, table and analytical step.

Figure-specific support is as follows. Fig. 1 is supported by the study-level construct table, evidence-layer definitions, source ledgers, model summaries and validation summaries. Fig. 2 is supported by service points, host-venue candidates, 500 m grid and OSM-derived spatial layers, district emergence-suppression indices and `data/figure_source_data/hssc_fig2_quality_v2_20260616/Figure_02_source_map.json`. Fig. 3 is supported by host-venue and rule-liminal candidate files, rule-source and semantic records, network and threshold outputs and the Fig. 3 panel source tables. Fig. 4 is supported by grid and district emergence-suppression outputs, sparse-rule repair files, controlled-model summaries, matched-frontier diagnostics and the Fig. 4 panel source tables. Fig. 5 is supported by the manual-validation files and validation panel source tables. Fig. 6 is supported by multilayer edge and topology files, threshold and simulation outputs, controlled-model outputs, morphology and grid features and Fig. 6 panel source tables. Fig. 7 is supported by case-selection records, panel-statistics files, rule ledgers, grid outputs, service and host layers and the Fig. 7 manifest. Tables 1-4 and Supplementary Table S1 are supported by the construct and evidence tables, empirical contrast summaries, source-class summaries, robustness screens and data-code availability table supplied in `tables/` and `data/`.

Raw commercial POI archives, raw full-text source captures and raw platform or web captures are not redistributed because they are subject to third-party licence, access and copyright conditions. Raw downloaded OSM packages are not redistributed in this package; the package instead provides derived OSM layers and source manifests, and the corresponding context can be reacquired under the original OSM licence terms. The provided derived coordinates, categories, grid-level outputs, source-class summaries, validation results and metadata are sufficient to evaluate the reported claims from the shareable derived layer. Before publication, the shareable derived data and documentation will be deposited in a DOI-issuing repository, with a double-anonymous repository link or supplementary file used during peer review if required by the journal.

Custom scripts supporting the analysis are provided in `code/analysis_scripts/` with path configuration and an input checker. Nine scripts run directly from the supplied derived data: scripts 34, 38, 40, 43, 46, 47, 48, 66 and 67. Three upstream scripts document restricted steps requiring licensed raw-derived intermediates: script 24 requires `hilton_shenzhen_structured_pet_policies_v1.csv`, script 33 requires `full_silent_rule_venues_2025_v1.csv`, and script 37 requires `pet_service_core_A_deduplicated.csv`. Figure-building scripts and panel source tables are supplied for review of figure assembly; final figure files are supplied separately in the manuscript submission package, while this repository provides the underlying panel source tables and manifests under `data/figure_source_data/`. The code package therefore supports direct rerunning of the reported numerical diagnostics from the shareable derived layer, while full raw-to-derived reconstruction requires access to the restricted source materials described above.
